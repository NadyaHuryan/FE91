1. В модулях всегда используется режим use strict. Например, присваивание к необъявленной переменной вызовет ошибку.
  2. Каждый модуль имеет свою собственную область видимости. Другими словами, переменные и функции, объявленные в модуле, не видны в других скриптах.
  3. Модули должны экспортировать функциональность, предназначенную для использования извне. А другие модули могут её импортировать.
  4. Код в модуле выполняется только один раз при импорте Если один и тот же модуль используется в нескольких местах, то его код выполнится только один раз, после чего экспортируемая функциональность передаётся всем импортёрам.
